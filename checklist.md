# checklist.md

# AI 视觉对话助手验收清单

## 1. 项目结构验收

* [ ] 项目根目录存在 `frontend/` 目录。
* [ ] 项目根目录存在 `backend/` 目录。
* [ ] 项目根目录存在 `README.md`。
* [ ] 项目根目录存在 `.env.example`。
* [ ] 项目根目录存在 `spec.md`。
* [ ] 项目根目录存在 `tasks.md`。
* [ ] 项目根目录存在 `checklist.md`。
* [ ] 执行 `find frontend/src -maxdepth 3 -type f | wc -l` 返回值大于等于 `8`。
* [ ] 执行 `find backend/app -maxdepth 4 -type f | wc -l` 返回值大于等于 `8`。

## 2. 启动验收

* [ ] 在 `backend/` 目录执行后端启动命令后，终端没有 Python 导入错误。
* [ ] 访问后端健康检查接口，返回内容中包含 `ok`。
* [ ] 在 `frontend/` 目录执行前端启动命令后，浏览器可以打开页面。
* [ ] 浏览器页面中能看到摄像头区域、聊天区域、角色区域。
* [ ] 浏览器控制台没有连续刷新的红色运行时报错。
* [ ] 前端 `.env` 或配置文件中能看到后端地址配置。
* [ ] 后端 `.env.example` 中包含模型 API Key 的占位字段。
* [ ] 后端 `.env.example` 中包含是否启用 Mock 模式的配置项。

## 3. 摄像头功能验收

* [ ] 页面存在「开启摄像头」按钮或等价入口。
* [ ] 点击开启摄像头后，浏览器弹出摄像头权限申请。
* [ ] 授权摄像头后，页面中能看到实时视频预览。
* [ ] 拒绝摄像头权限后，页面显示 `无法访问摄像头，请检查浏览器权限`。
* [ ] 未开启摄像头时，点击视觉问答入口显示 `请先开启摄像头`。
* [ ] 开启摄像头后，点击截图测试按钮，可以在页面或控制台看到截图成功提示。
* [ ] 执行 `grep -r "getUserMedia" frontend/src` 返回大于等于 `1` 条。
* [ ] 执行 `grep -r "canvas" frontend/src` 返回大于等于 `1` 条。
* [ ] 上传给后端的截图宽度不超过 `768px`。
* [ ] 上传给后端的截图格式为 `image/jpeg` 或 `image/webp`。
* [ ] 单张上传图片大小小于等于 `500KB`。

## 4. 语音输入验收

* [ ] 页面存在「开始说话」按钮或等价入口。
* [ ] 点击语音输入按钮后，AI 角色切换到 `listening` 状态。
* [ ] 用户说「你好」后，输入框或聊天区域出现识别文本 `你好`。
* [ ] 语音识别失败时，页面显示 `语音识别失败，请重试或使用文字输入`。
* [ ] 浏览器不支持语音识别时，页面显示 `当前浏览器不支持语音识别，请使用文字输入`。
* [ ] 页面存在文本输入框。
* [ ] 在文本输入框输入 `你好` 并发送后，聊天区域出现用户消息 `你好`。
* [ ] 语音输入和文本输入最终都会在聊天区域新增一条用户消息。
* [ ] 执行 `grep -r "recognition" frontend/src` 返回大于等于 `1` 条。
* [ ] 执行 `grep -r "listening" frontend/src` 返回大于等于 `2` 条。

## 5. 普通聊天验收

* [ ] 输入 `你好` 后，后端返回一条 AI 回复。
* [ ] 普通聊天请求不携带图片字段。
* [ ] 普通聊天过程中，角色状态先变为 `thinking`，收到回复后变为 `talking`。
* [ ] AI 回复显示在聊天区域。
* [ ] AI 回复文本长度大于 `0`。
* [ ] 后端普通聊天接口在 Mock 模式下也可以返回固定回复。
* [ ] 后端普通聊天接口返回字段中包含 `reply`。
* [ ] 后端普通聊天接口返回字段中包含 `emotion`。
* [ ] 后端普通聊天接口返回字段中包含 `action`。
* [ ] 输入 `今天天气怎么样` 时，如果没有接入天气工具，AI 不应编造实时天气，应提示自己无法直接获取实时天气或建议用户提供城市信息。
* [ ] 执行 `grep -r "chat" backend/app` 返回大于等于 `3` 条。

## 6. 视觉意图判断验收

* [ ] 输入 `你看看桌上有什么` 时，系统判定为需要视觉信息。
* [ ] 输入 `看一下我手里拿的是什么` 时，系统判定为需要视觉信息。
* [ ] 输入 `这个东西是什么颜色` 时，系统判定为需要视觉信息。
* [ ] 输入 `你好` 时，系统判定为不需要视觉信息。
* [ ] 输入 `给我讲个笑话` 时，系统判定为不需要视觉信息。
* [ ] 非视觉问题不会触发截图。
* [ ] 非视觉问题不会调用后端视觉接口。
* [ ] 视觉问题会触发截图。
* [ ] 视觉问题会调用后端视觉接口。
* [ ] 执行 `grep -r "看看\\|看一下\\|摄像头\\|画面\\|手里\\|桌上" frontend/src` 返回大于等于 `3` 条。

## 7. 视觉理解验收

* [ ] 开启摄像头后，输入 `你看看桌上有什么`，前端会上传当前截图。
* [ ] 后端视觉接口收到图片后返回视觉描述。
* [ ] 视觉描述字段长度大于 `0`。
* [ ] Mock 模式下，视觉接口返回文本包含 `我看到了`。
* [ ] 真实模型模式下，视觉接口返回与画面相关的描述。
* [ ] 当图片为空时，后端返回错误，不会导致服务崩溃。
* [ ] 当图片格式不支持时，后端返回错误，不会导致服务崩溃。
* [ ] 视觉描述会被拼接进后续聊天上下文。
* [ ] AI 最终回复中能体现画面信息，而不是只回答通用话术。
* [ ] 执行 `grep -r "vision" backend/app` 返回大于等于 `3` 条。
* [ ] 执行 `grep -r "image" backend/app` 返回大于等于 `3` 条。

## 8. 成本控制验收

* [ ] 系统默认不连续上传视频流。
* [ ] 页面停留 `30` 秒且用户不提问时，后端视觉接口调用次数为 `0`。
* [ ] 输入 `你好` 后，后端视觉接口调用次数仍为 `0`。
* [ ] 输入 `你看看桌上有什么` 后，后端视觉接口调用次数增加 `1`。
* [ ] 连续两次在 `10` 秒内输入视觉问题时，第二次优先复用缓存。
* [ ] 复用缓存时，后端日志出现 `reuse vision cache`。
* [ ] 非视觉问题后端日志不出现 `vision model called`。
* [ ] 视觉问题后端日志出现 `vision model called` 或 `reuse vision cache`。
* [ ] 上传图片宽度不超过 `768px`。
* [ ] 上传图片质量压缩后单张小于等于 `500KB`。
* [ ] 执行 `grep -r "cache" backend/app` 返回大于等于 `2` 条。
* [ ] 执行 `grep -r "MAX_IMAGE" backend/app frontend/src` 返回大于等于 `1` 条。
* [ ] 执行 `grep -r "VISION_CACHE" backend/app` 返回大于等于 `1` 条。

## 9. AI 角色动画验收

* [ ] 页面初始状态下，角色显示为 `idle`。
* [ ] 用户点击语音输入时，角色显示为 `listening`。
* [ ] 用户发送问题后，角色显示为 `thinking`。
* [ ] AI 回复返回并开始播放语音时，角色显示为 `talking`。
* [ ] 语音播放结束后，角色回到 `idle`。
* [ ] 角色状态变化可以通过页面文字、CSS class 或调试面板观察到。
* [ ] `idle` 状态有轻微待机动画。
* [ ] `listening` 状态有明显聆听提示。
* [ ] `thinking` 状态有加载或思考动画。
* [ ] `talking` 状态有说话动画。
* [ ] 执行 `grep -r "idle" frontend/src` 返回大于等于 `2` 条。
* [ ] 执行 `grep -r "listening" frontend/src` 返回大于等于 `2` 条。
* [ ] 执行 `grep -r "thinking" frontend/src` 返回大于等于 `2` 条。
* [ ] 执行 `grep -r "talking" frontend/src` 返回大于等于 `2` 条。

## 10. AI 回复语音播放验收

* [ ] AI 回复显示后，浏览器自动播放对应语音。
* [ ] 语音播放期间，角色保持 `talking` 状态。
* [ ] 语音播放结束后，角色回到 `idle` 状态。
* [ ] 浏览器不支持语音播放时，页面显示 `当前浏览器不支持语音播放`。
* [ ] 关闭语音播放开关后，AI 回复只显示文本，不播放声音。
* [ ] 执行 `grep -r "speechSynthesis" frontend/src` 返回大于等于 `1` 条，或者后端存在 TTS 服务调用代码。
* [ ] AI 回复文本为空时，不触发语音播放。
* [ ] 连续两次收到 AI 回复时，上一段语音会停止或排队，不会重叠播放。

## 11. 错误处理验收

* [ ] 后端不可用时，前端显示 `后端服务不可用，请稍后重试`。
* [ ] 模型调用失败时，后端返回可展示错误，不暴露 API Key。
* [ ] 摄像头未授权时，页面显示 `无法访问摄像头，请检查浏览器权限`。
* [ ] 麦克风未授权时，页面显示 `无法访问麦克风，请检查浏览器权限`。
* [ ] 视觉问题但没有摄像头画面时，页面显示 `请先开启摄像头`。
* [ ] 图片上传失败时，页面显示 `图片上传失败，请重试`。
* [ ] 语音识别失败时，用户可以改用文本输入完成对话。
* [ ] 任意一次接口失败后，页面不会白屏。
* [ ] 任意一次接口失败后，用户可以继续发送下一条消息。
* [ ] 浏览器控制台不出现未捕获 Promise 异常。

## 13. 端到端验收

* [ ] 启动后端服务。
* [ ] 启动前端服务。
* [ ] 浏览器打开前端页面。
* [ ] 点击开启摄像头，页面显示实时画面。
* [ ] 点击语音输入，说出 `你看看桌上有什么`。
* [ ] 页面识别出文本 `你看看桌上有什么`。
* [ ] 角色状态变为 `listening`，随后变为 `thinking`。
* [ ] 前端截取当前摄像头画面并上传。
* [ ] 后端日志出现 `vision model called` 或 `reuse vision cache`。
* [ ] 聊天区域出现用户消息 `你看看桌上有什么`。
* [ ] 聊天区域出现 AI 回复，且回复中包含画面相关内容。
* [ ] AI 回复开始语音播放。
* [ ] 语音播放期间角色状态为 `talking`。
* [ ] 语音播放结束后角色状态回到 `idle`。
* [ ] 再输入 `给我讲个笑话`。
* [ ] 第二次请求不上传图片。
* [ ] 第二次请求后端视觉接口调用次数不增加。
* [ ] 第二次 AI 回复正常显示。
* [ ] 整个流程中页面没有白屏。
* [ ] 整个流程中浏览器控制台没有未捕获异常。
* [ ] 整个流程可以在 `3` 分钟内向评审完整演示。

## 14. 最小交付判定

满足以下全部条件即可视为最小可交付：

* [ ] 摄像头可以打开并预览。
* [ ] 文本输入可以完成普通聊天。
* [ ] 语音输入可以转成文本并发送。
* [ ] 视觉问题可以触发截图。
* [ ] 后端可以返回视觉描述。
* [ ] AI 可以结合视觉描述回答。
* [ ] AI 回复可以显示在聊天框。
* [ ] AI 回复可以语音播放。
* [ ] 角色至少支持 `idle`、`listening`、`thinking`、`talking` 四个状态。
* [ ] 非视觉问题不调用视觉模型。
* [ ] `README.md`、`spec.md`、`tasks.md`、`checklist.md` 都存在。
* [ ] 至少完成一次端到端验收流程。
