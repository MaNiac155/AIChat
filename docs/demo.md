# 三分钟演示流程

## 演示前准备

1. 在根目录 `.env` 中设置 `MOCK_MODE=true`，或配置可用的真实聊天和视觉模型。
2. 启动后端并确认 `http://localhost:8000/api/health` 返回 `ok`。
3. 启动前端并打开 `http://localhost:5173`。
4. 确认浏览器允许摄像头、麦克风和声音播放。

## 固定演示步骤

### 1. 展示初始状态

- 页面显示摄像头、AI 角色和聊天区域。
- 角色状态为 `idle`。
- 说明摄像头仅在浏览器本地预览，不会持续上传视频。

### 2. 展示视觉语音问答

1. 点击“开启摄像头”，确认出现实时预览。
2. 开启“自动视觉理解”。
3. 点击“开始说话”，说出“你看看桌上有什么”。
4. 观察角色依次进入 `listening` 和 `thinking`。
5. 观察流程状态依次显示“正在理解当前画面”和“正在结合画面生成回答”。
6. 确认聊天区域显示用户问题、截图信息和 AI 回复。
7. 确认 AI 回复开始播放时角色进入 `talking`，播放结束后回到 `idle`。
8. 后端日志应出现 `vision model called` 或 `reuse vision cache`。

### 3. 展示成本控制

1. 在 5 分钟内再次提问视觉问题。
2. 页面应显示“复用视觉缓存”，后端日志出现 `reuse vision cache`。
3. 输入“给我讲个笑话”。
4. 确认普通聊天正常回复，并且视觉模型调用次数没有增加。

### 4. 展示错误恢复

1. 关闭摄像头。
2. 输入“你看看桌上有什么”。
3. 页面显示“请先开启摄像头”，且页面不白屏。
4. 随后输入普通问题，确认仍可继续聊天。

## 演示观察点

- 角色顶部徽标可观察 `idle`、`listening`、`thinking`、`talking` 状态。
- 对话输入区上方可观察当前主流程状态。
- 视觉反馈区域可观察截图大小、缓存复用状态和视觉模型调用次数。
- “回复语音”和“自动视觉理解”均可独立关闭。
